# Dictum — Naming Policy

Dictum's text is **CC BY 4.0** and its tooling is **MIT** (see [`LICENSE`](LICENSE))
— anyone may share, adapt, and translate it. This short policy is about the one
thing the licenses deliberately don't settle: use of the **name "Dictum."**

**This is not a registered trademark, and the project does not intend to register
one.** The name is claimed only by **first use** — being the first to publish a
standard under it — and this policy is stated so that intent is on the record. It
is a request to the community, **not a legal threat**, and the project is not
asserting or planning to enforce formal trademark rights.

## What actually keeps one authoritative "Dictum"

Not this policy — **provenance does.** The official standard is whatever is
published from the canonical repository, and each release is verifiable by a
signed tag + checksum (see [`RELEASES.md`](RELEASES.md)). That is the real
guarantee. The naming convention below is a courtesy that keeps the ecosystem
unconfused; it is not what makes a copy authentic.

## Please

- **Use the name freely to refer to the standard** — "documented to Dictum," "a
  Dictum-style doc set," "based on Dictum" — in articles, talks, tooling, and
  comparisons. No permission needed.
- **If you fork, adapt, or translate the text** (CC BY lets you), please give your
  version a **different name** and don't present it as the official Dictum
  standard. Crediting Dictum as the source is appreciated (CC BY asks for it anyway).
- **Please don't** imply official endorsement or a formal "Dictum-certified"
  conformance program — none exists.

## Conformance claims (interim)

Until any conformance process exists, describe your relationship **factually and
with a version** — for example *"documented to Dictum v1.0.0"* or *"Dictum-style
doc set"* — rather than as a certification.

## If the name has to change

The name is **not load-bearing.** If a conflict arises, the standard can be
renamed without touching the method, the contracts, or any product's docs — only
this cover label changes.

## Contact

For anything not covered here, open an issue on the project repository.
