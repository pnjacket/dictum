# Concern 11.5 — Quality & Testing

> **One-line:** What "tested" means for this product — the test strategy, the coverage map over capabilities and invariants, and the real-flow E2E standard. Defines; Delivery Process enforces.

## Purpose & Scope

Owns the **test strategy**, **quality bars**, the **coverage map**, and the **real-flow E2E standard**. It does **not** own the Definition of Done, vertical-slice rule, or build sequence (Delivery Process) — Delivery *references* this concern's E2E standard for its proof-of-done gate.

## Owned Contracts (referenced by stable ID)

1. **Coverage map** — maps **every in-scope contract ID** to the test(s) that cover it, across the interface/domain/product set — not just the request/REST contracts. Concretely: each in-scope `CAP-###` maps to its required test types (real-flow E2E for user-facing); each `INV-###` maps to an assertion; each `API-###` to a contract/integration test; each **`EVT-###`** on the event surface to a test of its **delivery, authorization, and fan-out** behavior (a distinct contract no `API-###` test covers — the live/push path is exercised, not inferred from the request path); and each product **`SUCCESS-###`** to the test that demonstrates it. Prefer **one `INV-###` per test** (or a separately-identifiable assertion per invariant) so a failure attributes to a single invariant. **Edge inputs, not only happy values:** every in-scope **input-bearing interface element** — an `API-###` parameter, a `CLI-###` flag, a `LIB-###` parameter, a form field — is tested at its **default and boundary inputs**: every optional input *absent* **and** *empty* (the value a UI submits for a blank/“all” control), plus one malformed value — asserting the contracted outcome (a valid result or the uniform error model, never a raw framework error). **An ID with no observable check gets an explicit `n/a` row stating why.** Some in-scope IDs are constitutionally untestable (a `PERSONA-###`, an advisory environment row, a factual boundary statement, an attestation); the map records the disposition rather than inviting a theater check — **silent omission is the defect**, the stated `n/a` is not. **Referenced by** Delivery Process (DoD checks coverage). The test-side complement to the traceability spine.
2. **Real-flow E2E standard** — `E2E-STANDARD`: a test exercises the system's **own** code paths for real (real authentication — no bypass / no token injection, real navigation, real backend logic) and asserts the real rendered outcome. **“Real navigation” means operating the screens, not merely reaching them:** the E2E **drives every interactive control** on each in-scope `SCREEN-###` it visits — every form/button (filter, run, search, sort, paginate, submit) — **at least once, including in its default/empty state**, and asserts a **non-error** rendered outcome. **External dependencies may be substituted by approved fakes per the environment's fidelity map** (owned by Operations): substituting an external (e.g. S3 → a container) is legitimate; bypassing own code (e.g. login) never is. The bar is thus *environment-relative*. Three further rules of the standard: **(a) expectation pre-computation via pure product modules is sanctioned** — a test may import the product's *pure* modules into the test process to compute expected values (e.g. deriving expected terrain or a feasible route from the product's own deterministic generator); this injects nothing into the system under test and is observation-adjacent, not a bypass. **(b) Determinism journeys assert run-invariant observables** — a journey claiming reproducibility must compare values that are identical across runs *by contract* (exact initial state, settled end-states, exact key/delta sets), never time-sampled trajectories. **(c) Policy-liveness checks bind to page-governed observables** — a check that a runtime-governed policy (e.g. a CSP) is *live* must observe what the **page** governs (injected-script non-execution, rejected requests, real violation events), never harness-context evaluation: automation harnesses run privileged and are typically exempt from the page's policy, so a harness-eval probe passes vacuously. **Referenced by** the delivery methodology's proof-of-done.
3. **Quality bars** — coverage thresholds, required test types, and gate definitions. Includes the **flake / re-run policy**: what a red run obligates (investigate; re-run scope), and whether retries/quarantine exist — recommended **zero retries, no quarantine list** for functional tiers; for *measurement* tiers the re-run rule is **co-defined with Performance & Scalability (11.13)**. The post-investigation disposition for a genuine transient (re-run green, zero code change) is **record the incident plus both runs** — never a quarantine list; a wall-clock-sensitive observation row may state an environment load-condition precondition instead.

## Maturity Ladder — what each rung means HERE

| Rung | Meaning for Quality & Testing |
|---|---|
| **Absent** | No test strategy. |
| **Sketch** | Which test types will be used, as bullets. |
| **Specified** | Test approach per type described; rough coverage goals; test-data approach noted. |
| **Contract-grade** | Coverage map over **every in-scope contract ID** — every `CAP-###` (incl. real-flow E2E for user-facing) + an assertion per `INV-###` + **edge/default inputs per input-bearing interface element** + **delivery/authz/fan-out per `EVT-###`** on the event surface + a demonstration per `SUCCESS-###`, with an explicit `n/a — why` row for any ID with no observable check; quality bars + required gates stated (incl. the flake/re-run policy); test-data strategy defined; `E2E-STANDARD` specified, **including operation of every in-scope screen's interactive controls (default state included)**. A team or AI can implement the tests without guessing what "tested" means. |
| **Verified** | *(delivery methodology)* Tests implemented and passing in CI. |

## Contract-grade bar

Coverage defined over **every in-scope contract ID** — per `CAP-###` (incl. real-flow E2E for user-facing) + an assertion per `INV-###` + **edge/default inputs per input-bearing interface element** (API param, CLI flag, LIB param, form field) + **delivery/authz/fan-out per `EVT-###`** (where an event surface exists) + a demonstration per `SUCCESS-###`, with an **explicit `n/a — why` row** for any in-scope ID that admits no observable check (silent omission fails the bar) — plus quality bars + a named **flake/re-run policy** (co-defined with 11.13 for measurement tiers) + test-data strategy + an `E2E-STANDARD` that **operates every in-scope screen's controls in their default state** (not just reaches the screens).
