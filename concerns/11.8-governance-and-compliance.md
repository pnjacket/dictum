# Concern 11.8 — Governance & Compliance

> **One-line:** Are we *allowed*, and are we *accountable* — the data-handling policy, regulatory obligations, audit requirements, and licensing/IP. Owns the *rules*; Security implements the *controls*.

## Purpose & Scope

Owns **all data-handling policy** plus regulatory compliance, audit policy, retention/residency, and license/IP. Completes the chain: **Domain tags** facts → **Governance** sets policy → **Security** implements mechanism (and **Observability** implements audit). The same chain runs with **Business & Legal (11.15) as the facts source**: where 11.15 owns a legal/IP constraint (e.g. a trademark clean-room rule), the **enforcement policy** realizing it is a Governance-owned `POLICY-###` — the legal constraint plays the role Domain's tags play for data. It does **not** own protection mechanisms or logging implementations.

## Owned Contracts (referenced by stable ID)

1. **Compliance policy** — `POLICY-###`. The prefix covers **compliance policy generally**; the canonical case is *data-handling* policy keyed to Domain's classification tags (protection level, retention, residency, consent), but an IP-compliance, legal-constraint-enforcement (facts owned by 11.15), or other regulatory policy carries the same prefix — no local re-mint needed. **Referenced by** Security & Privacy (implements the mechanism) and Domain & Data (lifecycle realizes retention).
2. **Audit policy** — what must be auditable + audit-log retention. **Referenced by** Observability & Monitoring (implements the trail).
3. **Framework obligation → control map** — each applicable obligation mapped to an owning concern/control. **Referenced by** those concerns.
4. **License/IP register (dependencies)** — inbound **dependency** licenses + ownership, cleared; **plus the product's own outbound license stance**, including a *deliberate absence* ("no license granted / all rights reserved") — the stance is a contract either way, not only the inbound column. This is the *dependency* boundary; first-party source is contract 5.
5. **First-party source-provenance register** — every non-trivial **vendored / adapted / ported / transliterated** unit of *first-party* source, attested `{origin, license, outbound-compatibility determination}`. Code copied under a license **incompatible with the outbound stance is a defect, not a smell**. A **green dependency license scan is not source-provenance clearance** — it is structurally blind to code copied into first-party source (no package, no import, no edge). Where code is **model-authored** (the `code_authorship` trait), provenance is **undeclared by default** — an LLM can reproduce licensed source with no attribution — so a **best-effort detection/attestation pass is owed**, not an assumption of "original". **Honest residual (Part 0.7 spirit):** undeclared regurgitation may still evade detection; the pass is best-effort and its residual is *recorded*, never papered over as "cleared". The `// SOURCE: <origin> <license>` marker (Std Part 5) is the code-site attestation this register consumes. **Referenced by** Quality & Testing (the provenance check) and Business & Legal (a declared don't-derive-from constraint mints a `LEGAL-###`).

## Consumed (referenced, not owned)

- Classification tags + `ENTITY-###` lifecycle (Domain & Data).
- Security mechanisms (realize the protection policy); Observability (realizes the audit policy).

## Maturity Ladder — what each rung means HERE

| Rung | Meaning for Governance & Compliance |
|---|---|
| **Absent** | No governance doc (only a truly throwaway tool). |
| **Sketch** | License/IP basis noted; obvious obligations flagged (the Baseline minimum). |
| **Specified** | Data-handling policy described; applicable frameworks named; audit needs noted. |
| **Contract-grade** | `POLICY-###` per classification tag; framework obligations mapped to owning controls; audit requirements + retention/residency stated; **inbound** license/IP cleared **and first-party source provenance attested** (each non-trivial vendored/adapted unit's origin + license + outbound-compat; detection run where code is model-authored, residual recorded). Checkable: every tag has a policy, every obligation an owner, every non-trivial first-party unit a provenance determination. Scoped to whatever regulation pulled in. |
| **Verified** | *(delivery methodology)* Controls in place and evidenced. |

## Contract-grade bar

Policy per classification tag + framework obligations mapped to controls + audit requirements + retention/residency + **inbound** license/IP cleared + **first-party source-provenance attested** (incompatible copy = defect; a green dependency scan is *not* source clearance; where code is model-authored a best-effort detection pass is run and its residual recorded — attestation alone reproduces the declaration-blindness it exists to close).
